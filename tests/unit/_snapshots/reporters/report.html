PSE-GH-001 — AI agent with cloud-credential grant on a fork-reachable event

Permission Slip Effect: a fork-reachable trigger, an AI agent action, and a cloud-credential grant co-exist. An attacker who steers the agent via prompt injection holds a valid OIDC token.

Fix: Break at least one leg of the triangle: 1. Gate the agent job by same-repo identity. 2. Drop the id-token: write grant from the agent job. 3. Narrow the agent's allowedTools to forbid shell.

Reference: https://securitylab.github.com/resources/github-actions-preventing-pwn-requests/

SEC2-GH-002 — Missing top-level permissions block

Workflow does not declare a top-level permissions block.

Fix: Add `permissions: contents: read` at the top level.

Reference: https://docs.github.com/en/actions/security-guides/automatic-token-authentication

SEC3-GH-001 — Action referenced by mutable tag

An external action is referenced by a mutable tag (e.g. @v4) instead of a 40-character commit SHA. An attacker who compromises the action's repo can move the tag to a malicious commit; every consumer pinned by tag picks up the new code on the next workflow run.

Fix: Pin the action to a full 40-character commit SHA. Use `gh action pin` or `taintly --fix`.

Reference: https://docs.github.com/en/actions/security-guides/security-hardening-for-github-actions
